Template Words

These are the words people use when talking about templates. Once these click, the rest of the guide is much easier to read.

Template File

The template.txt file is where every template entry lives. Save the file and EHR picks up the change next time it needs a template.

Tag

The tag is the name before the colon, like welcome or rules. Tags can use letters, numbers, and underscores.

Entry

An entry is one complete template block: the tag, optional bracket rules, the colon, and the message content.

Header

The header is the first line, shaped like tag[properties]: content. The brackets are optional.

Content

Content is the message EHR sends. It can be on the same line after the colon or on the lines below it.

Properties

Properties are bracket rules like map=Skeld, delay=5, hidden, or regex="^discord$".

Placeholder

A placeholder like {{PlayerName}} or {{RoomCode}} gets replaced with live game info when the message sends.

User Variable

A user variable is your own shortcut, like @discord = discord.gg/ehr. Use @discord later instead of typing the link again.

One tag can have multiple entries. That is how you make random choices, fallbacks, or different messages for different situations.
A hidden entry still works normally. It just does not show in the public /template list.
A regex trigger is different from a tag: the tag names the entry, while regex watches chat for text that should fire it.
What is the template file?

The template.txt file is the single text file where all your template data lives. This is the file you edit to add or change any behavior. 

<ModFolder>/EHR_DATA/template.txt

When you save changes, EHR picks them up the next time a matching event occurs. You usually do not need to restart the game just because you changed a template.

What is the difference between a tag, entry, and template?

A tag is the name you give to a piece of content so EHR knows how to find and send it. Tags appear at the start of a line, before the square brackets and colon. 

greet[hidden]: Hello!
^^^^^
tag

An entry is one complete block of content in the template file. It starts with a tag and colon, and its content is everything that follows until the next header appears.

Template is the general word people use for the whole system, or loosely for one entry or a group of entries. When someone says "write a template for X", they usually mean "write an entry, or set of entries, under a tag that handles X."

What are headers, content, and properties?

The header is the first line of an entry. It contains the tag, any properties, and optionally the start of the content. 

tag[properties]: content

Content is the actual message that gets sent when an entry is triggered. It is everything after the colon on the header line, plus any extra lines below it until the next header.

Properties are optional conditions written inside square brackets. They control when and how an entry behaves, like which map it applies to, how likely it is to be picked, whether it is hidden, or whether it requires a chat message to match a regex pattern. 

greet[map=Skeld, weight=3, hidden]: Hello from the Skeld!
Supported properties in plain English

Properties are optional conditions written inside the square brackets of a header. They control when and how an entry behaves, things like which map it applies to, how likely it is to be picked, or whether it requires a specific role.

map=Only send this entry on specific maps. Separate multiple maps with |, like map=Skeld|Polus.
meeting=Only send during a specific meeting number. Example: meeting=1 for the first meeting only.
role=Only send to players with a specific role. Prefix with ! to exclude a role, like role=!Impostor.
rank=Only send to players with a specific rank, such as Host, Admin, Mod, Moderator, or VIP.
players=Only send when the player count meets a condition, like players>=8 or players<=5.
preset=Only send when a specific game preset is active. Separate multiple presets with |.
weight=How likely this entry is to be picked when multiple entries share the same tag. Higher means more likely.
delay=Send this entry after a delay in seconds instead of immediately.
hiddenHide this tag from the public tag list. The entry still works normally.
regex=Only fire this entry when an incoming chat message matches the given pattern.

Multiple properties are separated by commas. Some properties are key=value pairs, while hidden is just a key by itself. Its presence alone is enough to activate it.

What are placeholders, variables, and comments?

A placeholder is a special token written inside the content that gets replaced with a live value when the message is sent. 

announce: There are currently {{PlayerCount}} players in the lobby!

A user variable is a custom shorthand you define yourself with @name = value. Once defined, you can write @name anywhere in your content and it gets substituted with the value you set. 

@server = Endless Host Roles Official

welcome: Welcome to @server!

A comment is a line EHR ignores. Start the line with # when you want to leave yourself a note or organize the file.

What are weighted picks, delayed entries, and cache?

When multiple entries share the same tag and at least one of them has weight=, EHR uses a weighted pick to randomly choose one entry to send instead of sending all of them. Higher weight means more likely to be chosen. 

tip[weight=3]:Common tip.
tip[weight=1]:Rare tip.

A delayed entry is an entry with delay=. Instead of sending immediately, it waits the specified number of seconds before the content is sent. 

welcome:This sends right away.
welcome[delay=10]:This sends 10 seconds later.

Cache means EHR keeps a parsed copy of the template file in memory so it does not re-read and re-parse the file on every single message. When the file changes, that parsed copy is rebuilt.

You do not usually need to think about cache. Just save your file and EHR will pick up the changes on the next relevant event.

Choose When It Sends

Some tag names are automatic. Other tags are only sent when you run them manually.

Automatic system tags are triggered by the game. You do not call them yourself.
I want a message when someone joins

Use the welcome tag.

welcome:Welcome {{PlayerName}}! Code is {{RoomCode}}.
I want a repeating lobby reminder

Use the notification tag. The repeat timing comes from the auto-message settings.

  • Enable Auto-Message toggles the feature on or off.
  • The Auto-Message Is Sent Every X seconds controls how often it fires.
OnFirstMeeting:This is the first meeting. Crewmates, use the /myrole command to see your role info.
notification:{{PlayerCount}} players in lobby.

That example sends one first-meeting reminder and one repeating lobby notification.

I want a meeting message

Use OnMeeting for every meeting, or OnFirstMeeting for only the first one.

OnFirstMeeting:This is the first meeting. Use /myrole if you need your role info.
OnMeeting[meeting=2]:Second meeting. Start narrowing it down.
I want a message I can run myself

Make your own tag name, then trigger it with /template [tagname].

rules:Please read the lobby rules before joining voice.

To send it, run /template rules or /t rules.

Add Rules With Brackets

Brackets are optional. Use them when a message should only send in a specific situation.

Basic property syntax: tag[property=value]:message.
Multiple properties are separated by commas: tag[property=value, property=value]:message.
A template with no properties still works exactly as before. Properties are fully optional.
How do I delay a message?

Add delay=N, where N is 1-30 seconds. Delayed messages always send.

welcome:This sends right away.
welcome[delay=5]:This sends 5 seconds later.
welcome[delay=15]:This sends 15 seconds later.
How do I pick one random message?

Use the same tag more than once. Add weight=N if one should show more often.

motd[weight=8]:Remember to vote every meeting.
motd[weight=2]:Rare tip: check who is avoiding tasks.

Weights can be 1-10. Higher weight means that entry gets picked more often.

How do I send only on certain maps?

Use map=X. Separate multiple maps with |.

Valid map values are Skeld, MiraHQ, Polus, Airship, Fungle, and Dleks.

mapinfo[map=Skeld]:You are on Skeld!
mapinfo[map=Polus|Fungle]:You are on Polus or Fungle.
How do I send only to certain players?

Use role for in-game roles or rank for Host, Admin, Mod, Moderator, or VIP.

strategy[role=Jester]:Act suspicious, but survive long enough.
strategy[role=!Impostor]:You are not an Impostor. Work together and find them.
strategy:A generic tip for anyone else.

access[rank=Mod|Admin|Host]:Welcome, prestigious member!

You can add a fallback template with no role or rank condition for everyone else.

How do I check lobby size, preset, or meeting number?

Use players, preset, or meeting.

Player count supports >, >=, <, <=, =, and !=.

lobbysize[players>=15]:Big lobby! Starting soon.
lobbysize[players<6]:Still waiting for more players to join.
welcome[preset=2|3]:This message appears on Preset 2 or 3.
OnMeeting[meeting=3|4]:Meeting 3 or 4. Time is running out.
How do I hide a tag from the list?

Use hidden. The tag will not show in the /template listing, but it can still trigger.

ping[hidden]:Just an unfinished hidden message.

The regex property watches incoming chat messages and fires a template entry automatically when the message matches. It is just a way to describe what a chat message should look like.

Template syntax: tag[regex="pattern"]:message.

EHR checks the raw chat message and ignores uppercase/lowercase differences.
Use ^ and $ when you want the whole message to match exactly.

What is regex?

Regex, short for regular expression, is a compact way to describe text. Instead of checking only one exact sentence, you can describe patterns like "contains hello", "starts with !rules", "has a number", or "says body as its own word."

For EHR templates, the pattern goes inside regex="...". When someone sends chat, EHR tests that chat against the pattern. If it matches, the template sends. 

rules[regex="^!rules$"]:Please read the lobby rules before the game starts.
Regex for dummies

Start with normal words, then add symbols only when you need more control.

hello Matches any message containing hello.
^!rules$ Matches exactly !rules and nothing else.
^!settings Matches messages that start with !settings.
thanks$ Matches messages that end with thanks.
body|report Matches body or report anywhere in the message.
^(hi|hello)$ Matches exactly hi or exactly hello.
what.*role Matches what, then anything, then role.
colou?r Matches color or colour.
\\bbody\\b Matches body as its own word, not somebody.
\\d+ Matches one or more numbers.
v1\\.0 Matches a literal dot, like v1.0.
How does EHR use it?

EHR reads every template with a regex property, compiles that pattern, then checks incoming chat with it. If the pattern matches and any other bracket rules also pass, EHR sends the template. 

help[regex="^!help$"]:Available commands: !rules, !discord, !settings

discord[regex="^!discord$"]:Join the Discord: discord.gg/maul

bodyhelp[regex="\\b(body|report|dead)\\b"]:
If you found a body, say where it was and who was nearby.

In practice, a player typing !discord triggers the Discord line. A player typing where was body? triggers the body help line.

If you want the simplest autoresponder, make the word people type the regex and make the text after the colon the answer. 

discord[regex="^discord$"]:discordlink

That means: if someone says exactly discord, EHR replies with discordlink. Replace discordlink with the real invite when you are ready.

Literal text and simple matches

The simplest pattern is just a word or phrase written out plainly. Spaces and punctuation are matched literally unless they have a special meaning in regex. 

rulecheck[regex="what are the rules"]:The rules are posted in the Discord!

By default, a pattern can match anywhere inside the message. So regex="hello" matches hello, say hello to me, and hello everyone.

Quantifiers: how many times?

Quantifiers go right after whatever they apply to and control how many times it repeats. They are how you say "optional", "one or more", or "exactly this many".

*Zero or more times. hel*o matches heo, helo, and hello.
+One or more times. hel+o matches helo and hello, but not heo.
?Zero or one time. colou?r matches both color and colour.
{4}Exactly 4 times. \\d{4} matches exactly 4 digits.
{2,8}Between 2 and 8 times.
{3,}3 or more times.

The combination .* is extremely common. It means "anything at all in between". 

roleinfo[regex="what.*role"]:Check the role list with /roles!
Anchors: start and end of a message

Anchors pin a pattern to a specific position. Use them when you want to avoid accidental matches.

^Match must be at the very start of the message.
$Match must be at the very end of the message.
 
help[regex="^!help$"]:Here is the help menu.
settings[regex="^!settings"]:This fires when the message starts with !settings.
thanks[regex="thanks$"]:You are welcome!

Without anchors, regex="yes" can also fire on words like yesterday and yessir. Anchors give you control.

Character sets: one of these characters

Square brackets define a set of acceptable characters at one position. The pattern matches if any one character from the set is found there.

[aeiou]Any one vowel.
[a-z]Any lowercase letter.
[A-Z]Any uppercase letter.
[0-9]Any digit.
[a-zA-Z]Any letter, either case.
[a-zA-Z0-9_]Any letter, digit, or underscore.
 
gray[regex="gr[ae]y"]:Gray and grey both work.
code[regex="^[A-Za-z]{6}$"]:That looks like a 6-letter room code.

Put ^ as the first character inside the brackets to flip the meaning. For example, [^0-9] means "anything except a digit".

OR, groups, and repeated text

Use | to say "this or that". Parentheses group parts of a pattern together so the choices share the same anchors or repeat as one unit. 

greet[regex="\\b(hi|hello|hey|sup|yo)\\b"]:Welcome to the lobby!

This catches any of those greeting words. Grouping matters: ^hi|hello$ means "starts with hi OR ends with hello". Use ^(hi|hello)$ when you mean the entire message is exactly hi or exactly hello. 

laugh[regex="(ha)+"]:That is a lot of laughing.
good[regex="(very )*good"]:Glad to hear it.
Dots, numbers, words, and shortcuts

A dot . matches any single character: a letter, digit, symbol, space, anything. Shortcuts describe common kinds of characters.

.Any single character.
.*Anything at all in between.
\\dAny digit, same as [0-9].
\\wAny word character: letters, digits, or underscore.
\\sAny whitespace, like a space or tab.
\\bA word boundary, useful for matching a word by itself.
 
roleinfo[regex="what.*role"]:Check the role list with /roles!
pin[regex="^\\d{4}$"]:That looks like a 4 digit PIN.
mention[regex="@\\w+"]:That looks like a player tag.
Inline flags

Flags go at the very start of your pattern inside (?...) and change how the pattern behaves. EHR already ignores uppercase/lowercase differences, but these flags are useful to recognize when testing or sharing patterns.

(?i)Case-insensitive. hello matches Hello, HELLO, and HeLLo.
(?m)Multiline. ^ and $ match the start and end of each line.
(?s)Dot-all. . can also match newline characters.
(?x)Extended. Spaces in the pattern are ignored and # starts a regex comment.
 
greet[regex="(?i)^\\b(hi|hey|hello|sup)\\b"]:Welcome to the lobby!

You can combine flags freely, like (?im). Most EHR templates will not need flags, but knowing what they are makes examples from regex tools easier to read.

Escaping special characters

Some characters have special meaning in regex: . * + ? ^ $ { } [ ] | ( ) \. If you want to match one of them literally, put a backslash in front of it.

A literal dot\\.
A literal question mark\\?
Literal parentheses\\(sus\\)
Three dots\\.\\.\\.

This is one of the most common mistakes: v1.0 means v1, then any character, then 0. Use v1\\.0 when you mean the real dot in v1.0.

How do I combine regex with other rules?

Regex can sit next to other bracket rules. Separate properties with commas. This example only responds to !settings on Preset 2 or 3. 

settings[regex="^!settings$", preset=2|3]:
Current setup: {{Map}} | {{PlayerCount}} players | Preset {{Preset}}

You can also hide regex-triggered templates so they do not appear in the manual /template list. 

rules[regex="^!rules$", hidden]:
No slurs, no teaming, no stream sniping, and keep meetings readable.
Useful real examples

These are ready to copy and adjust.

# Respond to greetings
greet[regex="\\b(hi|hello|hey|sup|yo)\\b"]:Hey! Welcome to the lobby!

# Answer role questions
roleinfo[regex="what.*(role|roles)"]:Check the role list with /roles in chat!

# Share the room code when asked
roomcode[regex="(room ?code|what.s the code|how (do i|to) join)"]:The room code is {{RoomCode}}!

# Catch a message that is only 6 letters, like an Among Us room code
SelfPromo[regex="^[A-Za-z]{6}$"]:DO NOT ADVERTISE YOUR LOBBY HERE!

# Reply when someone types only a number
countdown[regex="^\\d+$"]:Numbers are not votes. Use the meeting buttons.

Common mistakes
  • Use quotes around the regex value: regex="^!rules$".
  • Double backslashes in regex shortcuts: write \\d+, not \d+.
  • Escape dots when you mean a real dot: write v1\\.0, not v1.0.
  • Use ^ and $ for commands so random sentences do not trigger them.
  • Group shared choices with parentheses: use ^(hi|hello)$, not ^hi|hello$.
  • Use | for "or", such as body|report|dead.
  • Keep broad patterns like .* rare because they match almost everything.

Testing tip: regex101.com is useful if you set the flavor to .NET. Remember to double backslashes again when you paste the final pattern into template.txt.

Testing your patterns

Before adding a pattern to your template file, test it at regex101.com and set the flavor to .NET. Paste in sample messages to see what matches and what does not. The site also explains what each part of your pattern does, which is great for learning.

One EHR-specific reminder: this template parser stores doubled backslashes. If regex101 shows \bdiscord\b, write \\bdiscord\\b in template.txt.

No matter how careful you are, broad patterns can still catch messages you did not expect. Test with examples that should match and examples that should not match.

Reuse Text With Variables

Variables are handy for links, lobby summaries, and repeated lines you do not want to type over and over.

How do I make a variable?

Put the variable on its own line, then use it later with @name.

@discord = discord.gg/ehr
@session = {{Date}} | {{Time}}
@lobby = {{Map}} | {{PlayerCount}} players | Code: {{RoomCode}}

welcome:
@lobby
Join our server at @discord!

Placeholders inside variables are filled when the message sends.

What should I avoid?
  • Do not define variables in the middle of a message.
  • Do not put comments at the end of a template line.
  • Start comments on their own line with #. Comment lines are ignored entirely.
  • If a variable is not defined, it stays as-is in the final message.
# This is a comment
welcome: Greetings!

Troubleshooting

If a template does not send the way you expected, check these first.

It does not show in /template

Make sure the tag is not marked hidden. Hidden templates can still trigger, but they do not show in the list.

The message does not send

Check the bracket rules. A map, role, rank, player count, preset, or meeting condition may not be met.

A variable appears as text

Make sure the variable is defined on its own line, like @discord = discord.gg/ehr.

Delay feels different than expected

Immediate entries go through weighted picks. Delayed entries always fire after their delay.
