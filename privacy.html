What The App Does

Modded Among Us Games is a private Discord community bot used to support and automate the Modded Among Us Lobbies server.

Community Management

The bot helps staff and members with moderation, role management, verification-related workflows, giveaways, events, minigames, Mafia and Among Us community tools, host tracking, warning systems, member support, server notifications, and activity tracking.

Server-Only Use

The bot is intended for use within Modded Among Us Lobbies and related community workflows. It is not a public multipurpose bot and is not used to sell user data or serve advertising.

Information We Collect

The bot only collects or processes Discord information needed for its enabled features.

Discord Account And Server Data

  • Discord user IDs, usernames, display names, and mentions.
  • Server IDs, channel IDs, thread IDs, role IDs, message IDs, and application IDs.
  • Member role information and server join dates when needed for permissions, panels, or eligibility checks.
  • Invite-related information when needed for member access or invite tracking workflows.

Bot Feature Data

  • Giveaway entries, eligibility checks, bonus entries, reroll records, and giveaway audit logs.
  • Moderation records such as warnings, mutes, punishments, staff notes, role changes, and audit entries.
  • Game, event, Mafia, host, and participation statistics.
  • Economy and minigame data, including balances, inventory, rewards, cooldowns, participation, and transaction logs.
  • User opt-in preferences for role panels, notifications, pings, stream alerts, and related features.
  • PingMe / DCD lobby notification settings, subscriber lists, forum thread settings, and optional thread emoji suffixes.

How We Use Information

Operational Use

Information is used to run commands, manage roles, enforce server requirements, support moderation, operate giveaways, track game hosting, run minigames, manage server notifications, and maintain server configuration.

Safety And Abuse Prevention

Information may also be used to prevent spam, duplicate entries, cheating, command abuse, moderation evasion, or misuse of server systems.

Discord Gateway Intents

The bot uses Discord gateway intents so Discord can send events needed for commands, moderation, role systems, notifications, and lobby tools. The bot uses Discord's default intents plus the privileged message content, server members, and presence intents.

Message Content Intent

Used when message text is needed for prefix commands, custom commands, autoresponders, sticky messages, Mafia or game log parsing, chat activity tracking, economy rewards, moderation tools, and other chat-based server features.

Server Members Intent

Used to read member information needed for verification-related workflows, role panels, role management, permission checks, moderation actions, warning systems, eligibility checks, participation tracking, and member support tools.

Presence Intent

Used for features that depend on Discord activity status, especially Among Us rich presence. This lets the bot detect when a tracked host appears to be in an open lobby, post lobby-open messages, ping opted-in members, and show a user's current activity details through debug commands.

Default Discord Events

Used for normal server operation, such as knowing which servers, channels, threads, roles, messages, reactions, voice states, invites, emojis, and interactions exist where the bot has permission. These events support commands, panels, logs, reaction features, voice or lobby tools, and server automation.

Message Content

Why Message Content Is Used

The bot may read message content in server channels where it has access when message content is required for enabled features. This includes prefix commands, custom commands, autoresponders, sticky messages, Mafia or game log parsing, chat activity tracking, economy rewards, and chat-based minigames.

Limits

Message content is used for bot functionality only. It is not sold, used for advertising, or shared with unrelated third parties.

Presence And Activity Data

Lobby Notifications

The bot may use Discord presence or activity data for features that detect relevant game or lobby states, such as Among Us lobby reopen notifications. This allows opted-in members to receive useful pings when a tracked lobby becomes available.

Activity Debugging

Presence information may also be shown to a user through debugging commands so they can verify whether Discord is exposing the activity needed for lobby notification features.

PingMe And Lobby Data

The PingMe / DCD lobby tools use forum thread, subscriber, and presence information to notify members when a host's Among Us lobby reopens.

Stored Configuration

The bot may store server IDs, enabled forum channel IDs, forum thread IDs, thread owner user IDs, whether DCD pings or lobby-open messages are enabled, subscriber user IDs waiting for a ping, and optional thread emoji suffixes used when a thread is renamed.

Presence Information Used

When a tracked post owner has Discord activity available, the bot may read activity type, activity name, state, and details. For Among Us activity, this can include whether the player is "In Lobby," a lobby code, region, and host/lobby label exposed by Discord rich presence.

Runtime And Output

The bot may temporarily track recent lobby codes, requested thread names, cooldowns, and send queues to avoid duplicate or rate-limited notifications. PingMe output may mention subscribed members, post a lobby-open message, include the lobby code, or rename a forum thread using the lobby code, host name, region, and configured emoji suffix.

Cleanup

Subscriber lists are cleared after a DCD reopen ping is sent or when a user removes themselves. Thread state may remain while the feature is enabled for that post, while subscribers are waiting, or while related thread settings are still active.

Storage And Access

Where Data Is Stored

Bot data may be stored in local databases, configuration files, logs, or other storage used to operate the bot. Stored data may include moderation records, giveaway records, economy data, game statistics, server settings, and user opt-in preferences.

Who Can Access It

Access is limited to authorized server staff, bot maintainers, or administrators who need the information to operate, moderate, troubleshoot, or maintain the bot and server.

Retention And Deletion

Retention

Data is kept only as long as needed for bot functionality, server administration, moderation history, audit logs, giveaways, game statistics, or abuse prevention.

Deletion Requests

Users may request deletion of eligible stored data. Some data may be retained when needed for moderation history, safety, unresolved disputes, audit logs, or abuse-prevention requirements.

Sharing

No Sale Of Data

We do not sell user data. We do not share collected data with advertisers or unrelated third parties.

Normal Server Visibility

Some bot output may be visible in server channels as part of normal functionality, such as leaderboards, giveaway entries, public stats, role panels, event posts, or game participation information.

Contact

For privacy questions, data requests, bot-related concerns, or priority support, contact the server owner, admin email, or Priority Tickets channel.
